# AGENTS.md

## Название проекта
Городской

## Правила работы

1. Перед началом любой сессии читай этот файл целиком, а также все файлы, указанные в разделе "Обязательные для чтения" ниже.
2. После реализации каждой задачи или функционала - пиши тесты и прогоняй ВСЕ предыдущие тесты вместе с новыми. Если что-то сломалось - чини прежде, чем двигаться дальше.
3. Работай аккуратно: ничего не ломай из того, что уже реализовано и протестировано.
4. После каждого крупного блока работы обновляй этот файл: что сделано, где лежит, какие ключевые решения приняты.
5. До тех пор, пока все задачи из `task.md` не будут реализованы и приняты владельцем проекта, в начале каждой сессии обязательно читай `project-requirements.md` и `task.md`. После полной реализации и приемки всех задач из `task.md` эти файлы можно перестать подгружать как обязательный контекст, если они не нужны для текущей работы.

## Чеклист после создания или изменения важных документов

После создания или существенного изменения ТЗ, архитектурного документа, плана задач, схемы данных, правил безопасности или другого опорного документа обязательно проверь:

1. Нужно ли добавить документ в раздел "Обязательные файлы для чтения".
2. Нужно ли обновить историю изменений в этом файле.
3. Не появились ли новые решения, ограничения или команды запуска, которые должны быть явно зафиксированы для следующих сессий.
4. Если документ влияет на разработку или приемку, он должен быть добавлен в обязательные файлы для чтения сразу в том же блоке работы.

## Обязательные файлы для чтения

1. `project-requirements.md` - техническое задание MVP справочного сервиса официальных ответов городской администрации.
2. `task.md` - пронумерованный план автономных этапов реализации MVP по техническому заданию.
